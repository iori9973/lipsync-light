# LipSync Light

VRChat アバターのリップシンクパラメーター（`Voice` / `Viseme`）に連動して、マテリアルのエミッション（発光色）を制御する Animator 設定を自動生成する Unity Editor 拡張です。

口を動かすと光る目、声に合わせて輝くアクセサリーなど、リップシンク連動の演出を手軽に実現できます。

**Modular Avatar を使った非破壊ワークフロー**で動作するため、既存の FX Controller を直接書き換えません。

---

## 動作環境

- Unity 2022.3.22f1 以上
- VRChat SDK（VRCSDK3-AVATAR）3.5.0 以上
- **Modular Avatar 1.10.0 以上**（必須）

---

## インストール

### VCC（VRChat Creator Companion）経由（推奨）

1. VCC の **Settings → Packages → Add Repository** を開く
2. 以下の URL を入力して「Add」

```
https://iori9973.github.io/lipsync-light/index.json
```

3. プロジェクトの Manage Packages から **LipSync Light** を追加
   - Modular Avatar も依存関係として自動でインストールされます

## 基本的な使い方

### 1. ウィンドウを開く

Unity メニューバーの **Tools → LipSync Light** を選択します。

### 2. Avatar Root を設定する

**Avatar Root** フィールドに、アバターのルートとなる GameObject を Hierarchy からドラッグ&ドロップします。

Avatar Root を設定すると、アバターの Hierarchy 配下に **`LipSync Light`** という GameObject が自動生成されます。この GameObject を無効にすると LipSync Light の効果をまとめてオフにできます。

### 3. モードを選択する

| モード | 内容 |
|---|---|
| **Voice 連動** | `Voice` パラメーター（0〜1）の大きさに応じてエミッション強度をブレンド。声が大きいほど明るく光る |
| **Viseme カラー** | `Viseme` パラメーター（0〜14）の口の形に合わせてエミッションカラーを切り替える |
| **両方** | Voice 連動と Viseme カラーを同時に使用する |

### 4. ターゲットを追加する

**[+ ターゲットを追加]** ボタンで光らせたい Renderer を追加します。

各ターゲットには以下を設定します：

| 項目 | 説明 |
|---|---|
| **Renderer** | 光らせたいメッシュの Renderer コンポーネント |
| **Material** | 対象のマテリアルインデックス（複数マテリアルがある場合にドロップダウンで選択） |
| **Property** | 発光を制御するシェーダープロパティ名（複数選択可） |

#### プロパティの選択

Renderer を設定すると、シェーダーの発光関連プロパティがチェックボックス形式で表示されます。**複数のプロパティを同時選択**して、1 回のセットアップでまとめて制御できます。

- 既知の発光プロパティ（`_EmissionColor` など）は日本語ラベル付きで先頭に表示
- シェーダーにある他の発光系プロパティも自動検出して表示
- リストにないプロパティは「手動追加」テキストフィールドから直接入力できます

### 5. カラーを設定する

モードに応じて、各ターゲットのカラー設定が表示されます。

#### Voice 連動モードのカラー設定

| 項目 | 説明 |
|---|---|
| **消灯** | 無音時（Voice = 0）のエミッションカラー。通常は黒（発光なし） |
| **点灯** | 発声時（Voice = 1）のエミッションカラー |

#### Viseme カラーモードのカラー設定

「Viseme カラー詳細」を展開すると、口の形（0〜14）ごとのカラーを個別に設定できます。

| Viseme | 口の形 | 音の例 |
|---|---|---|
| 0: sil | 無音（口を閉じた状態） | — |
| 1: PP | 唇を閉じる | p, b, m |
| 2: FF | 上歯が下唇に触れる | f, v |
| 3: TH | 舌が歯の間に | th |
| 4: DD | 舌先が上歯茎に触れる | d, t, n |
| 5: kk | 舌の奥が上顎に触れる | k, g |
| 6: CH | 舌が上顎近くに上がる | ch, j, sh |
| 7: SS | 歯を閉じ気味にする | s, z |
| 8: nn | 鼻に抜ける | n, ng |
| 9: RR | 舌を丸める | r |
| 10: aa | 大きく開く | a |
| 11: E | 横に広げる | e |
| 12: ih | やや開く | i |
| 13: oh | 丸く開く | o |
| 14: ou | 唇を突き出す | u |

> `0: sil`（無音）は話していないときに常にアクティブです。光らせたくない場合は黒（発光なし）のままにしてください。

#### 切り替え時間（Viseme モード）

**切り替え時間 (秒)** を 0 より大きい値にすると、口の形が変わったときに前の色から新しい色へグラデーションのように変化します。

## Voice 設定

Voice 連動モードでは以下の詳細設定が可能です。

| 項目 | 説明 |
|---|---|
| **強度倍率** | 点灯色の明るさを増幅する倍率（1.0 = 等倍、2.0 で 2 倍の明るさ） |
| **点灯閾値** | Voice がこの値を超えると点灯を開始する（0〜1、デフォルト 0.1） |
| **フェード時間** | 点灯・消灯の切り替えにかかる時間（秒）。0 だと即時切り替え |

---

## 加算ブレンドモード

**「既存の発光に加算する」** を有効にすると、Animator レイヤーが加算（Additive）モードで動作します。

これにより、アバターにすでに設定されている定期的な発光アニメーションなどと**競合せず共存**できます。加算モードでは消灯色を黒にしておくと、無音時は他のアニメーションに一切影響を与えません。

## カラーグループ機能

複数のターゲットで同じカラー設定を共有したい場合に使用します。

### グループの作成方法

**方法 A：「この設定でグループを作成」ボタン**

ターゲットのカラーを設定した後、ターゲットカード下部の **[この設定でグループを作成]** ボタンを押すと、現在の設定が新しいグループとして登録され、そのターゲットが自動でグループに切り替わります。

**方法 B：「+ グループを追加」ボタン**

カラーグループセクションの **[+ グループを追加]** で空のグループを作成し、手動でカラーを設定します。

### グループの使い方

各ターゲットのカラードロップダウンから使用するグループを選択します。グループのカラーを変更すると、そのグループを参照しているすべてのターゲットに即座に反映されます。

> Voice と Viseme は別々にグループを設定できます。例えば「Voice は全ターゲット共通、Viseme はターゲットごとに個別」といった設定も可能です。

## セットアップ実行

**[セットアップ実行]** ボタンをクリックすると以下が自動生成されます：

- アバター Hierarchy 配下の `LipSync Light` GameObject に `MA Merge Animator` コンポーネントを設定
- `Assets/LipSyncLight/LipSyncLight_FX.controller`（専用の AnimatorController）
- `Assets/LipSyncLight/Animations/` 以下に AnimationClip

生成された FX Controller は Modular Avatar がビルド時にアバターの FX にマージします。

---

## 生成物の削除

**[生成物を削除]** ボタンで、生成した AnimationClip・FX Controller および MA Merge Animator コンポーネントをまとめて削除できます。

## よくある質問

**Q. セットアップ実行後、VRChat にアップロードするだけで動きますか？**
A. はい。Modular Avatar がビルド時に自動でマージするため、通常通りアップロードすれば動作します。

**Q. 既存の発光アニメーション（周期的に光るなど）と共存できますか？**
A. 「既存の発光に加算する」を有効にすると加算ブレンドモードで動作し、既存のアニメーションを上書きせずに共存できます。消灯色を黒にしておくと無音時は影響ゼロです。

**Q. 再セットアップすると既存の設定はどうなりますか？**
A. 専用の FX Controller を上書き再生成します。アバター本体の FX Controller には手を加えません。

**Q. lilToon / Poiyomi で使えますか？**
A. 使えます。Renderer を選択するとエミッション関連プロパティ（`_EmissionColor` など）がチェックボックス形式で自動表示されます。

**Q. Voice 連動で声の強弱に応じて光り方が変わりますか？**
A. はい。`Voice` パラメーターは 0〜1 の float 値で、閾値を超えるとフェードインし、下回るとフェードアウトします。On 状態では Blend Tree により声の強さに応じて明るさが連続変化します。

**Q. Viseme カラーモードで声を出していないのに光ります。**
A. `0: sil`（無音）の Viseme カラーが黒以外に設定されているためです。常時光らせたくない場合は `0: sil` のカラーを黒にしてください。

**Q. v1.x からアップグレードするには？**
A. v1.x で生成した FX レイヤーは自動移行されません。v1.x の「生成物を削除」を実行してから v2.0 でセットアップをやり直してください。
